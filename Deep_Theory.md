# 3. Most Common Interview Questions

# HTML & CSS Questions

## Q1. Difference between inline, block, and inline-block?

| Type | Description |
|---|---|
| Inline | Takes only required width |
| Block | Takes full width |
| Inline-block | Allows width and height |

---

## Q2. What is Flexbox?

Flexbox is used for:
- Alignment
- Spacing
- Responsive layouts

---

## Q3. Difference between Grid and Flexbox?

| Flexbox | Grid |
|---|---|
| One-dimensional | Two-dimensional |
| Row OR column | Rows AND columns |

---

# JavaScript Questions

## Q1. Difference between let, const, and var?

| Keyword | Scope | Reassign |
|---|---|---|
| var | Function | Yes |
| let | Block | Yes |
| const | Block | No |

---

## Q2. What is Closure?

A closure is a function that remembers variables from its outer scope.

---

## Q3. What is Hoisting?

JavaScript moves variable and function declarations to the top before execution.

---

## Q4. Difference between == and ===?

```txt
==  → checks value
=== → checks value and datatype
```

---

# React Questions

## Q1. What is React?

React is a JavaScript library used for building user interfaces.

---

## Q2. What are Hooks?

Hooks are functions that allow state and lifecycle features in functional components.

### Important Hooks
- useState
- useEffect
- useContext
- useRef

---

## Q3. What is Virtual DOM?

A lightweight copy of the real DOM used for faster UI updates.

---

## Q4. What is useEffect?

Used for:
- API calls
- Side effects
- Lifecycle handling

---

## Q5. What are Props?

Props are data passed from parent to child components.

---

## Q6. What is State?

State is dynamic data managed inside a component.

---

| React | React Native |

## Q2. What is Expo?

Expo is a framework/toolchain used for React Native development.

## Q3. How do you connect frontend with backend?

# 5. Deployment Strategy for Mobile & Web

# Frontend Deployment

## Netlify

Used for:
- React applications
- Portfolio websites
- Static frontend hosting

### Features
- Free hosting
- Continuous deployment
- GitHub integration
- Custom domains

---

# Frontend Deployment Steps

```txt
1. Push frontend code to GitHub
2. Login to Netlify
3. Import repository
4. Add build command
5. Add publish directory
6. Deploy website
```

---

# Backend Deployment

## Replit

Used for:
- Node.js backend hosting
- API deployment
- Cloud development

### Features
- Online IDE
- Instant deployment
- Easy API testing
- Beginner friendly

---

# Backend Deployment Steps

```txt
1. Upload backend project to Replit
2. Install dependencies
3. Add environment variables
4. Run backend server
5. Copy deployed API URL
6. Connect frontend APIs
```

---

# Deployment Architecture

```txt
Frontend (React / Netlify)
          ↓
Backend API (Node.js / Replit)
          ↓
Database (MySQL)
```
